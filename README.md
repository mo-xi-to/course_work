# Анатомический ассистент FMA

Интеллектуальная система на базе YandexGPT и онтологии Foundational Model of Anatomy (FMA). Проект предназначен для проведения глубокого анатомического анализа и логического вывода на основе графовых структур данных.

## Основные возможности
- Гибридный поиск: идентификация сущностей по названиям и FMAID.
- Многошаговое рассуждение: агент способен строить цепочки до 15 шагов для выявления неявных связей.
- Алгоритм обратного поиска: автоматическое восполнение разреженных данных через сканирование дочерних структур.
- Медицинская терминология: формирование ответов на русском языке с обязательным использованием латыни.

## Структура проекта
- `llm_agent.py` - ядро системы: логика многошагового агента и взаимодействие с Yandex Cloud API.
- `fma_db.py` - модуль данных: реализация алгоритмов прямого и обратного поиска в CSV-датасете.
- `tools.py` - слой инструментов: 5 специализированных интерфейсов для доступа ИИ к базе данных.
- `logger_config.py` - централизованная конфигурация логирования (уровни INFO, DEBUG, ERROR).
- `config.py` - управление путями и техническими параметрами моделей.
- `instructions.txt` - внешний слой системных инструкций.
- `app.py` - точка входа для веб-интерфейса на базе Gradio.
- `benchmark.py` - скрипт для автоматизированной оценки точности и скорости работы.

## Установка и запуск

### 1. Подготовка окружения
Требуется Python 3.10 или выше.
```bash
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

### 2. Настройка ключей
Создайте файл `.env` в корне проекта и добавьте

```
YANDEX_KEY=ваш_ключ
YANDEX_FOLDER_ID=ваш_id_каталога
```

### 3. Запуск 
- Веб-интерфейс: `python app.py` (доступен по адресу http://127.0.0.1:7860)
- Консольный режим: `python llm_agent.py`
- Тестирование: `python benchmark.py`
